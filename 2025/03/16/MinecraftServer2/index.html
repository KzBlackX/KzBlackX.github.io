Minecraft插件服务端搭建及配置 二

2025年03月16日 1.5k 字 大概 7 分钟

在上一章,我们已经编译好spigot核心并正确配置服务器参数后成功开服，此时还没有添加插件以及配置权限组，本章主要描述权限组的配置

添加插件

在 https://www.spigotmc.org/resources/ 上可以搜索你需要的插件，在spigot上，有不少免费插件，但是许多好用的插件是需要付费的，购买后的插件可以在 https://www.spigotmc.org/resources/purchased 上找到，而曾经浏览过的插件可以在 https://www.spigotmc.org/resources/watched 上找到。

注意！所有资源均由个人作者而非 SpigotMC 负责。访问权限可能随时被取消

以前，Essentials为最广泛实用的权限组插件，但是现在LuckPerms更简易实用，此文章将基于LuckPerms来配置权限组

LuckPerms下载和权限组配置

在 https://luckperms.net/download 下载所需要文件，我们本次的spigot需要下载对应的官网上的Bukkit插件。

将下载好的jar文件放入plugins文件夹中，启动服务器。

确认插件是否正常启用

配置权限组

LuckPerms支持在网页上配置权限组，这比使用指令配置更简单

  • 所有群组和在线/非默认用户： /lp editor或者lp editor all
  • 所有团体： /lp editor groups
  • 所有在线且非默认的用户： /lp editor users
  • 全体在线用户： /lp editor online
  • 对于特定群体： /lp group <group> editor
  • 对于特定用户： /lp user <user> editor
  • 仅针对具有特定权限的用户： /lp editor users <filter>或者lp editor <filter>

常规命令

用于操作 LuckPerms 功能的常规命令

用户命令

用于查看或修改特定用户的命令

/lp user <user> …由-组成，其中<user>是被查询/修改的用户的用户名或 uuid

组命令

用于查看或修改特定组的命令

/lp group <group> …由-组成，其中<group>是正在查询/修改的组的名称

权限命令

用于查看或修改特定用户或组的权限数据的命令

由以下/lp user <user> permission …任一形式构成/lp group <group> permission …

父命令

用于查看或修改特定用户或组的继承属性（父级）的命令

由以下/lp user <user> parent …任一形式构成/lp group <group> parent …

元命令

用于查看或修改特定用户或组的元数据的命令

由以下/lp user <user> meta …任一形式构成/lp group <group> meta …

追踪命令

用于查看或修改特定曲目的命令

/lp track <track> …由-组成，其中<track>是正在查询/修改的轨道的名称
