Minecraft插件服务端搭建及配置 一

2025年03月16日 4.3k 字 大概 18 分钟

准备工作

下载Minecraft服务端

本教程以创建此文章时最新核心1.21.4来搭建服务器，其他版本在某些细节可能不一样，但大体相同。

自1.8版本以后，spigot不再提供直链下载spigot服务端核心，需要用到 BuildTools 工具来编译spigot核心，此工具支持Windows/Linux/Mac。该工具的工作原理是下载 Vanilla Minecraft Server Jar、Bukkit、CraftBukkit 和 Spigot。然后，它会反编译 vanilla server jar、注入 craftbukkit 代码、应用 spigot 补丁，然后生成一个可用的 spigot jar 文件。

以前，BuildTools 是一个仅限命令行的工具，但自 2024 年 1 月 15 日起，BuildTools 现在带有 GUI，以帮助那些难以使用命令行的人。

需要一些先决条件程序，但以下说明将指导您完成所有需要做的事情

使用 BuildTools 需要两个应用程序：Git和Java

最新版本的 BuildTools 将自动为您下载并安装 Git。如果您遇到问题，请手动获取它！

下载 Git

Windows

Windows版本的BuildTools现在有GUI了！直接下载图形化安装程序构建spigot jar。

注意：BuildTools.exe需要使用管理员权限运行

Linux

可以通过包管理器使用单个命令安装 Git 和 Java。

  • Debian ： sudo apt-get install git openjdk-8-jdk-headless

  • Ubuntu：sudo apt install git openjdk-8-jdk-headless

  • CentOS/RHEL：sudo yum install git java-1.8.0-openjdk-devel

  • Arch：sudo pacman -S jdk8-openjdk git

下载 Java

  • Minecraft 1.17 以下： - 下载 Azul Zulu JDK 8

  • Minecraft 1.17[.1]： - 下载 Azul Zulu JDK 16

  • Minecraft 1.17.1 以上： - 下载 Azul Zulu JDK 17

  • 高于 Minecraft 1.20.5： - 下载 Azul Zulu JDK 21

下载并安装完所有应用程序后，建议设置一个文件夹供 BuildTools 运行。BuildTools 在空目录中运行效果最佳。

例如，您可能希望将 BuildTools 放在以下目录中：

在 Windows 上：

  • 如果使用cmd.exe： %USERPROFILE%\Downloads\BuildTools

  • 如果使用PowerShell： ${Env:USERPROFILE}\Downloads\BuildTools

  • 或者： C:\BuildTools

在 Linux/Mac 上：

  • ${HOME}/Downloads/BuildTools

注意： BuildTools 不能在 OneDrive 或 Dropbox 目录中运行！确保您创建的目录不位于其中一个目录中

启动服务端

将下载好的java文件放在根目录

创建启动脚本，Windows为run.bat , Linux为run.sh

运行脚本，第一次启动会下载服务器文件并提示 You need to agree to the EULA in order to run the server. Go to eula.txt for more info. 需要关闭服务器，将根目录下eula文件内eula=false改成eula=true,保存并关闭

以下是server.properties文件,此文件为MC的服务器设置文件，按需更改配置项，true代表执行，false代表不执行。

配置好所有文件，重启启动脚本，服务器补全库文件生成世界后提示Done (s)! For help, type "help"即开服完成！服务器根目录下world为主世界存档，world_nether为下界存档，world_the_end为末地存档，与Vanilla存档文件的对应关系为

  • world -> region 主世界文件夹
  • world_nether -> DIM-1 下界文件夹
  • world_the_end -> DIM1 末地文件夹
  • advancements 进度文件夹
  • data 数据包文件夹
  • playerdata 玩家数据

若要添加插件，则在根目录下plugins内添加对应的插件

至此，Minecraft插件服务端搭建及配置第一部分完！下一部分将解释添加插件后，使用权限组插件来支持给用户/用户组配置插件的指令权限。由于插件服务器的特殊性，即使是普通玩家也有使用指令的需求，而MC的默认配置导致插件的指令普通玩家无法使用。配置权限组可以在玩家没有管理员权限的情况下，仍然拥有所需的指令的使用权限，甚至可以拆分管理员权限指令，并将某些管理员指令的权限给予特定用户组的玩家。
